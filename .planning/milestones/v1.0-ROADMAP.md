## Overview

## Milestone Summary

**Key Accomplishments:**
1. Full Neon Strip redesign — all 4 pages rebuilt with shared `ui/` primitives (Button, Card, Tab, Badge); NightRiver brand (logo SVG, favicon, web manifest)
2. 20-animal avatar system — random on signup, user-changeable in Profile, live-propagating to all seats; 20 WebP assets shipped
3. Reconnect-safe sessions — 30s/120s grace windows, full GameState snapshot on reconnect, crash recovery at server boot, atomic buy-in/cashout (no double-spend)
4. Hidden admin panel — `/admin` Socket.io namespace with JWT auth, 4 live dashboards, 6 admin mutations, write-before-commit audit trail
5. 204-test Vitest + RTL suite — per-element coverage for all 11 interactive components, 5 scenario flows, GitHub Actions CI gate on push/PR to main
6. Sentry + PostHog observability with PII scrubbing; ToS/Privacy/RG compliance pages + server-side joinTable enforcement

**Key Decisions:**
- Design language locked: Neon Strip (tokens in CLAUDE.md)
- Deploy infrastructure explicitly OUT OF SCOPE for this cycle (deferred to v1.1+)
- Real-money payments OUT OF SCOPE (Deposit is a stub)
- Avatar system replaces Telegram avatar (20 generated anthropomorphic-animal images)
- Test stack: Vitest + React Testing Library, one file per interactive element
- Key-by-telegramId refactor was the linchpin — landed in Phase 1, unlocked reconnect/admin/history/analytics
- Dev-auth bypass hardened in Phase 1 (fail-closed gate + boot assertion + timingSafeEqual)
- D-09 species list LOCKED: fox, wolf, bear, tiger, panda, raccoon, lion, rabbit, owl, eagle, flamingo, penguin, crocodile, chameleon, cobra, shark, octopus, dolphin, frog, bat
- Admin auth uses HS256 JWT (8h) + ADMIN_TELEGRAM_IDS env allowlist — no isAdmin DB flag

**Known Gaps (accepted as tech debt):**
- BRAND-02 partial: raster favicon.ico and logo-192.png not generated (no tooling in executor env); SVG logo is the real in-app deliverable; Telegram users unaffected
- BRAND-01 partial: "bot handle references" means the actual Telegram @BotName — external Telegram configuration, not code

**Issues Deferred to v1.1+:**
- Deploy infrastructure: Dockerfile, nginx, HTTPS, CI/CD
- Real-money payments / KYC
- Street-by-street hand replayer
- 60-minute session "take a break" reminder
- Avatar unlock / streak rewards
- Telegram bot push notifications
- Self-hosted PostHog migration
- Admin password+cookie auth upgrade (from env allowlist)

**Technical Debt Incurred:**
- `player.avatarUrl` flagged `// DEPRECATED` in `types/index.ts` — cleanup deferred to a future pass after all consumers removed
- Profile History tab stub in 02-06 replaced by real data in Phase 3 (done)
- WebP Vite build warnings were resolved by shipping 20 WebP files (done)
