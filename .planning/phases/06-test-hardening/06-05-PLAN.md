  <action>
    Create new file `.github/workflows/ci.yml`. The directory may not exist yet; create it.

    Required content:

    ```yaml
    name: CI

    on:
      push:
        branches: [main]
      pull_request:
        branches: [main]

    jobs:
      build-and-test:
        name: Build & Test
        runs-on: ubuntu-latest

        steps:
          - name: Checkout repository
            uses: actions/checkout@v4

          - name: Setup Node.js
            uses: actions/setup-node@v4
            with:
              node-version: '22.x'
              cache: 'npm'
              cache-dependency-path: |
                package-lock.json
                client/package-lock.json

          - name: Install root dependencies
            run: npm ci

          - name: Generate Prisma client
            run: npx prisma generate

          - name: Install client dependencies
            run: npm ci
            working-directory: client

          - name: Build server (TypeScript compile)
            run: npm run build

          - name: Build client (Vite production)
            run: npm run build
            working-directory: client

          - name: Run tests (server + client)
            run: npm test
    ```

    Notes:
    - Uses `npm ci` (not `npm install`) for deterministic CI installs.
    - `prisma generate` runs after root install so the generated client exists before TypeScript compile of `server/db/`.
    - Uses Node 22.x to match the runtime engine STATE.md notes posthog-node prefers (22.22 ideal; 22.19 warning is non-blocking).
    - Works against `main` only â€” feature branches without a PR don't burn CI minutes.
    - No deployment steps. No Docker. No secrets required.

    If the repo's package-lock.json files don't exist (npm install was used instead of npm ci), the cache step will warn but not fail. The `npm ci` step will fail if no lockfile exists; in that case, the executor should switch `npm ci` to `npm install` for both install steps. Read the repo state â€” if `package-lock.json` exists at root, use `npm ci`; otherwise `npm install`.
