# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — project uses hand-rolled Neon Strip primitives |
| Preset | not applicable |
| Component library | Custom `ui/` primitives: Button, Card, Tab, Badge (Phase 2 substrate) |
| Icon library | none — emoji glyphs used for status icons |
| Font | System default (Telegram WebApp inherits Telegram's system font stack) |

No shadcn. The project has an established `ui/` primitive layer keyed to `tokens.ts` → `neon.css` CSS custom properties. All new components in this phase consume those primitives and tokens exclusively. No hex literals in component source.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Avatar ring gap, badge inner padding, micro separators |
| sm | 8px | Intra-card gaps, bubble label horizontal padding, card grid gap |
| md | 16px | Card padding default, section padding, tab content padding |
| lg | 24px | Empty-state vertical rhythm, section breaks within Profile |
| xl | 32px | Page vertical margins |
| 2xl | 48px | Not used in Phase 3 UI surfaces |
| 3xl | 64px | Not used in Phase 3 UI surfaces |

Exceptions:
- Action bubble pill padding: `4px 8px` (4px vertical / 8px horizontal — standard multiples of 4, keeps the pill compact over the seat card).
- History row expand region internal gap: `12px` (between board cards and hole cards — not a standard token; used only in this one place).
- ActionBubbleLayer z-index: `30` (above seat z-index of 20, below chat overlay).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 | 1.5 |
| Label | 11px | 700 | 1.4 |
| Heading | 14px | 700 | 1.3 |
| Display | 16px | 700 | 1.2 |

**Two weights only:** 400 (regular) and 700 (bold). No other weights permitted in Phase 3 component code.

Notes (source: existing SeatsDisplay.tsx, ProfileSettings.tsx):
- Bubble action text: 11px, weight 700, uppercase, letter-spacing 0.05em — maps to Label role. Matches `Badge` primitive sizing.
- Bubble amount suffix: 11px, weight 700, same line — space-separated after verb (e.g. `CALL 100`).
- Hand history row: date/table label 11px weight 700 (Label role), net delta 13px weight 700 monospace (Body role with monospace + color + glow providing visual distinction without a unique weight), board cards rendered via existing `Card` component at reduced scale.
- History empty-state heading: 14px weight 700 (Heading role).
- Section labels inside expanded HandHistoryRow (`BOARD`, `YOUR CARDS`, `SHOWN AT SHOWDOWN`): 11px, weight 700 (Label role), uppercase, letter-spacing 0.08em — mapped from 9px to 11px to stay within the 4-size scale.
- Stack/chip amounts always use `fontFamily: 'monospace'` and `fontVariantNumeric: 'tabular-nums'` — established project convention.

## Component Contracts

### ActionBubble (new — `client/src/components/ActionBubble.tsx`)

A single animated pill displayed over a seat.

**Focal point:** The active ActionBubble (z-index 30, neon glow, above all seat elements) dominates the GameRoom during its display window. Viewer attention tracks the acting seat by color — fold is red, call/check is cyan, raise is amber, all-in is orange.

**Structure:** Pill-shaped span with Neon Strip styling matching the `Badge` primitive recipe.

**Visual anatomy:**
```
┌─────────────────────────────────┐
│  [FOLD]  or  [CALL 100]  or ... │  ← uppercase action verb + optional amount
└─────────────────────────────────┘
  border: 1.5px solid <action-color at 50% alpha>
  background: rgba(10,10,14,0.88) + backdrop-blur(12px)
  box-shadow: 0 0 8px <glow-token>, inset 0 0 4px <glow-token>
  border-radius: 999px (pill)
  padding: 4px 8px
  font: 11px / 700 / uppercase / letter-spacing 0.05em
  color: <action-color CSS var>
  text-shadow: 0 0 6px <glow-token>
  white-space: nowrap
```

**Action → tier mapping (from `tokens.ts` ActionTier):**
| Action payload value | Tier | Color token | Glow token |
|----------------------|------|-------------|------------|
| `fold` | `fold` | `--color-action-fold` | `--glow-fold` |
| `check` | `call` | `--color-action-call` | `--glow-call` |
| `call` | `call` | `--color-action-call` | `--glow-call` |
| `bet` | `raise` | `--color-action-raise` | `--glow-raise` |
| `raise` | `raise` | `--color-action-raise` | `--glow-raise` |
| `allIn` | `allin` | `--color-action-allin` | `--glow-allin` |

**Label copy (exact strings):**
| action | amount present | Label |
|--------|----------------|-------|
| fold | — | `FOLD` |
| check | — | `CHECK` |
| call | yes | `CALL {amount}` |
| bet | yes | `BET {amount}` |
| raise | yes | `RAISE TO {amount}` |
| allIn | yes | `ALL-IN {amount}` |
| allIn | no (0) | `ALL-IN` |

Amount is the `amount` field from `ActionBubbleEvent` payload. Format: integer, no thousands separators (chips values at these stakes are ≤ 5 digits).

**Animation — `motion/react` `AnimatePresence`:**

Normal motion:
- Enter: `initial={{ scale: 0.8, opacity: 0 }}` → `animate={{ scale: 1, opacity: 1 }}` — duration 120 ms, ease `easeOut`.
- Hold: 900 ms minimum (D-04) — enforced by `useEffect` timeout before dequeue.
- Exit: `exit={{ opacity: 0, y: -6 }}` — duration 200 ms, ease `easeIn`.

`prefers-reduced-motion` (D-06): detect via `window.matchMedia('(prefers-reduced-motion: reduce)')`. If true:
- Enter: `initial={{ opacity: 0 }}` → `animate={{ opacity: 1 }}` — duration 0 ms (instant snap, no scale, no drift).
- Exit: `exit={{ opacity: 0 }}` — duration 0 ms.
- Hold: 900 ms unchanged — bubble is still displayed, action is still readable.

### ActionBubbleLayer (new — `client/src/components/ActionBubbleLayer.tsx`)

Mounts once inside `GameRoom`. Absolutely positioned layer covering the entire table area at `z-index: 30`.

**Positioning:** Each bubble is positioned using the same `SEAT_POSITIONS_DESKTOP` / `SEAT_POSITIONS_MOBILE` arrays already in `SeatsDisplay.tsx`. The bubble anchors to `{ left: pos.left, top: pos.top }` with the same `transform: pos.align` translate, then applies an additional upward offset:
- Standard seats: `translateY(-100%) translateY(-8px)` — places bubble above the seat card top edge, 8px gap.
- "My seat" mobile expanded layout (seat index == mySeat, isMobile): `translateY(-100%) translateY(-48px)` — accounts for the large card area rendered above the info strip.

Rotation offset (viewer at bottom) is applied identically to `SeatsDisplay`'s `visualIndex = (i - rotationOffset + 6) % 6` computation.

**State shape:** `Map<seatIndex, ActionBubble[]>` where each queue is a FIFO array. Per-seat: the head bubble renders; when it expires (900 ms hold + exit animation ~200 ms = 1100 ms total before dequeue), the next in queue renders. Queues are independent — no global serialization.

**Pointer events:** `pointerEvents: 'none'` on the layer. Bubbles must not intercept touch/click events on seats.

### HandHistoryList (new — `client/src/components/HandHistoryList.tsx`)

Rendered inside the Profile → History tab, replacing the Phase 2 stub.

**Focal point:** The net delta value (color + glow + monospace) anchors the eye on each HandHistoryRow. Green `+N` for wins, red `-N` for losses — legible at a glance in the collapsed list without expanding any row.

**Container:** `display: flex; flex-direction: column; gap: 8px` — 8px between rows. No horizontal padding (Card primitive provides 16px padding on the outer wrapper).

**Loading state:** While socket request is in-flight, render a neutral Card with centered `--color-neutral` text: `"Loading hand history..."` (13px, weight 400).

**Empty state:** Neutral Card with vertical stack (gap 14px, center-aligned):
- Icon area: 56px circle, `1.5px dashed` neutral border at 50% alpha, `♠` glyph at 28px in `--color-neutral` — replaces the Phase 2 timer emoji (same size/position contract from Phase 2 stub preserved).
- Heading: `"No hands yet"` — 14px weight 700, white.
- Body: `"Your played hands will appear here."` — 13px weight 400, `--color-neutral`, `line-height: 1.5`, `max-width: 280px`.

**Error state:** Fold-tier Card with centered text: `"Could not load hand history."` followed by `"Try closing and reopening your profile."` on a second line — 13px, `--color-action-fold`.

### HandHistoryRow (new — `client/src/components/HandHistoryRow.tsx`)

One Card primitive per hand, initially collapsed. Tap to expand.

**Collapsed state:**
```
┌──────────────────────────────────────────────────┐
│  [date label]     [table name]     [net delta]   │
│  11px neutral     11px neutral     13px mono      │
│  "3h ago"         "Standard 10/20"  "+250" green  │
│                                    "-100" red      │
└──────────────────────────────────────────────────┘
```
- Card variant: `neutral` (collapsed), `active` (expanded — cyan border).
- Card padding: 12px.
- Row layout: `display: flex; justify-content: space-between; align-items: center`.
- Date: relative time string (e.g. `"3h ago"`, `"2m ago"`, `"yesterday"`) — 11px, weight 700, `--color-neutral`.
- Table name: 11px, weight 700, `--color-neutral`. Truncated with `overflow: hidden; text-overflow: ellipsis; max-width: 100px`.
- Net delta: 13px, weight 700, monospace, `fontVariantNumeric: 'tabular-nums'`. Positive → `--color-action-sit` with `text-shadow: 0 0 6px var(--glow-sit)`. Negative → `--color-action-fold` with `text-shadow: 0 0 6px var(--glow-fold)`. Prefix: `+` for wins (always explicit), `-` for losses. Visual distinction from Label-role text is achieved via monospace + color + glow, not a unique weight.
- Full tap target on the row card (`cursor: pointer`, `active:scale-[0.99]`).

**Expanded state (tap to toggle):**
Reveals below the collapsed row content, separated by a `1px solid rgba(176,190,197,0.12)` divider:
```
  Board:  [5 board cards rendered as mini Card images, scale 0.55]
  Your cards: [2 hole cards, scale 0.55]  (only own cards — always shown)
  Opponents' cards: [shown only when showedDown=true — privacy rule D-18]
```
- Board and hole cards use the existing `HandDisplay` component with `size` prop set so cards render at approximately 32px wide each.
- Label above board: `"BOARD"` — 11px, weight 700, `--color-neutral`, uppercase, letter-spacing 0.08em (Label role).
- Label above hole cards: `"YOUR CARDS"` — 11px, weight 700, `--color-action-call` color for own cards (Label role).
- Opponents' cards section rendered only when `showedDown === true` for that row. Label: `"SHOWN AT SHOWDOWN"` — 11px, weight 700, `--color-neutral` (Label role).
- Expand/collapse animation: `motion/react` height animation `initial={{ height: 0, opacity: 0 }}` → `animate={{ height: 'auto', opacity: 1 }}` — 180 ms, easeOut. Honors `prefers-reduced-motion` (instant if reduced).
- Gap between board/cards sections: 12px.

**Result badge:** Pill Badge to the right of date — `variant="sit"` for win (`WIN`), `variant="fold"` for loss (`LOST`), `variant="neutral"` for chop (`CHOP`).

## Interaction States

### ActionBubble lifecycle

```
Event received
  └─ Enqueue to per-seat queue (Map<seat, Bubble[]>)
      └─ If seat queue was empty: immediately render head bubble
          └─ After 900ms hold + 200ms exit: dequeue, render next (if any)
      └─ If seat queue was non-empty: wait for head to expire, then render
```

Touch / reduced-motion:
- `prefers-reduced-motion: reduce` → no scale, no y-drift, instant in/out. Hold unchanged at 900 ms.
- No tap interaction on bubbles (pointer-events: none on layer).

### HandHistoryRow expand/collapse

- Tap anywhere on collapsed row Card → expand (Card border switches to `active`/cyan variant).
- Tap anywhere on expanded row Card → collapse.
- Only one row can be expanded at a time — expanding a new row collapses the previously expanded row.
- Expansion state is local to the `HandHistoryList` component (single `expandedHandId: string | null` state).

### History tab data loading

- Tab becomes active → `socket.emit('getHandHistory')` fires immediately.
- Response `handHistoryData` event → renders list (or empty state if `rows.length === 0`).
- Socket error or no response within 5 seconds → renders error state.
- No pull-to-refresh in Phase 3. Reload on tab re-enter (socket emit fires again when tab is re-selected).

## Accessibility

- ActionBubbleLayer: `aria-live="polite"` region so screen readers announce bubbles. Each bubble has `role="status"` with text content matching the label.
- HandHistoryList: `role="list"`, each HandHistoryRow is `role="listitem"`.
- HandHistoryRow expand button: `aria-expanded={isExpanded}`, `aria-label="Hand from {date} at {table}"`.
- Board and hole card sections: `aria-label="Board cards"` / `aria-label="Your hole cards"`.
- Touch targets: entire HandHistoryRow card is the tap target (min 44px height enforced by Card + 12px padding + row content).
