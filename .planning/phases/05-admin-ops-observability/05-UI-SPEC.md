## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding, input icon offset |
| sm | 8px | Inline form element spacing, button group gaps |
| md | 16px | Default panel padding, form field gap, card body padding |
| lg | 24px | Section padding, dashboard card grid gap |
| xl | 32px | Page-level side padding, dashboard row gap |
| 2xl | 48px | Major section breaks between dashboards |
| 3xl | 64px | Top offset below "ADMIN MODE" banner on scroll views |

Exceptions:
- ADMIN MODE banner height: 44px (matches the established 44px touch-target floor; also acts as a visual anchor line)
- Table/user action row height: 52px (accommodates two-line label + badge without wrapping)
- Input height: 44px (mobile tap target minimum, per CLAUDE.md)

## Component Inventory

All admin screens reuse the existing `client/src/components/ui/` primitives. New admin-specific compositions are built from these.

### Reused unchanged
- `Button` — variant prop from ActionTier union. Login uses `variant="active" emphasis`. Destructive actions use `variant="fold" emphasis`. Neutral/cancel uses `variant="neutral"`.
- `Card` — dark translucent panel. Dashboard sections use `variant="neutral"`. Active-table cards use `variant="active" glow`.
- `TabBar` + `Tab` — admin navigation (Tables | Users | Economy | Audit Log). Active tab = cyan underline GlowBar. Inactive = `var(--color-neutral)`.
- `Badge` — status pills. Table enabled = `variant="active"`. Table disabled = `variant="neutral"`. User banned = `variant="fold"`. Drain in progress = `variant="raise"`.

### New admin-specific compositions (built from primitives, no new primitives)

**AdminBanner** — permanent top bar, fixed position, z-index above all content:
- Background: `var(--color-action-raise)` (#ffab00) solid (not translucent — must be fully opaque for legibility)
- Height: 44px
- Text: "ADMIN MODE" — 13px, weight 700, uppercase, 0.08em letter-spacing
- Color: `#0a0a0e` (surface-base dark on amber — sufficient contrast, ~7:1)
- Width: 100vw, full bleed
- No close button — permanent while authenticated in /admin/*

**AdminLoginPage** — full-page dark surface, vertically centered form:
- Background: `rgba(10,10,14,0.90)` with `background: var(--color-surface-base)` fallback (no blur — top-level page)
- Form container: Card primitive, `variant="neutral"`, padding 24px, max-width 360px, centered
- Username input: 44px height, 1.5px border `color-mix(in srgb, var(--color-active) 50%, transparent)`, background `rgba(255,255,255,0.04)`, borderRadius 10px, 14px font, focus border upgrades to 100% opacity cyan
- Password input: same as username input
- Input label: 13px, weight 700, `var(--color-neutral)`, margin-bottom 4px
- Submit: Button `variant="active" emphasis fullWidth`, text "Sign In"
- Error message: 13px, `var(--color-action-fold)`, rendered between form fields and submit button
- No "forgot password" — single-credential MVP

**StatCard** — economy and summary number displays:
- Card primitive `variant="neutral"`, padding 16px
- Label: 13px, `var(--color-neutral)`, weight 700, uppercase
- Value: 28px, weight 700, color `var(--color-chip)` for chip counts, `var(--color-active)` for user/table counts
- textShadow glow matching the value color

**TableRow** — live table list entry:
- Height: 52px, flex row, gap 8px, padding 8px 12px
- Columns: table name (14px, white, truncated) | player count (13px, cyan) | blinds (13px, amber) | status Badge | action buttons
- Enable/Disable: Button `variant="active"` (enabled state) / `variant="neutral"` (disabled state), text "Enable" / "Disable"
- Drain: Button `variant="fold"`, text "Drain" — triggers confirmation inline
- Separator: 1px `color-mix(in srgb, var(--color-neutral) 15%, transparent)` border-bottom

**UserRow** — active user list entry:
- Height: 52px, flex row, gap 8px, padding 8px 12px
- Columns: display name (14px, white) | status Badge (seated/standing) | chip count (13px, amber) | action buttons
- Kick: Button `variant="fold"`, size compact (padding 4px 12px)
- Ban: Button `variant="fold" emphasis`, size compact — more prominent than Kick to distinguish severity
- Balance delta input: compact inline input (44px height, 80px width), accepts negative values; prefix display: green for positive, red for negative
- Grant button: Button `variant="active"`, text "Apply"

**BalanceDeltaInput** — inline signed integer input inside UserRow:
- Width: 80px, height: 36px (compact, not a primary CTA, 36px acceptable within a row of 52px)
- Placeholder: "±0"
- Value prefix: "+" rendered in `var(--color-action-sit)` green if positive; "−" in `var(--color-action-fold)` red if negative; no prefix at 0
- Validation: integer only, zod schema `.number().int().min(-100000).max(100000).refine(n => n !== 0)`

**AuditLogRow** — last-10 entries in Audit Log tab:
- 3-row micro layout: [action label] | [target + before→after] | [admin + timestamp]
- Action: 13px, weight 700, color determined by action type (kick/ban = fold-red; grant = sit-green; enable/disable/drain = raise-amber; edit = active-cyan)
- Before/after: 12px, monospace, `var(--color-neutral)` at 70% opacity
- Timestamp: 12px, monospace, `var(--color-neutral)` at 60% opacity
- Separator: 1px `color-mix(in srgb, var(--color-neutral) 12%, transparent)` border-bottom

**recharts usage** — Economy dashboard only:
- Background: transparent (inherits Card surface)
- Grid lines: `color-mix(in srgb, var(--color-neutral) 15%, transparent)`
- Bar fill: `var(--color-chip)` (#ffab00) with 80% opacity
- Tooltip: dark surface `rgba(10,10,14,0.95)`, 1.5px border `var(--color-active)`, 13px font, white text
- No legends (labels rendered above bars at 12px neutral)

## Screen-by-Screen Interaction Contracts

### /admin/login
- No session → show login form (no redirect loop)
- Submit with empty fields → zod client validation, inline field error "Required" below each field, no server call
- Submit with wrong credentials → server returns 401 → error message "Invalid username or password. Check your credentials." rendered above submit button; form is NOT cleared (username preserved, password cleared)
- Submit with correct credentials → JWT stored to localStorage, redirect to /admin/tables
- Already has valid JWT in localStorage → redirect immediately to /admin/tables (no flash of login form)
- JWT expiry (8h): any admin socket event returning 401/403 → clear localStorage JWT, redirect to /admin/login with message "Session expired. Sign in again."

### /admin/* (shell, authenticated)
- AdminBanner visible on all admin pages at all times, fixed top
- Body content has `paddingTop: 44px` to clear the banner
- TabBar below banner: tabs = Tables | Users | Economy | Audit Log
- Socket.io /admin namespace connected on mount; disconnected on unmount or logout
- Socket connection error → inline amber banner below AdminBanner: "Connection lost. Attempting to reconnect…"
- No link back to player UI — admin intentionally isolated

### /admin/tables (Tables tab)
- On connect: receive full `adminState` snapshot — renders immediately
- Live updates: `tableStateChanged` events update individual rows without full re-render
- Empty state (0 tables — not possible with 6 predefined tables; guard anyway): "No tables configured."
- Edit blinds/buy-in: inline form appears in-row on "Edit" click. Fields: "Small blind", "Big blind", "Buy-in". Submit = Button `variant="active"` "Apply (next hand)". Cancel = Button `variant="neutral"` "Cancel". Validated with zod: positive integers only, big blind = 2× small blind enforced with `.refine()`.
- Drain confirmation: inline row expands to show "Drain this table? Current hand will finish. No new seats after." — Button `variant="fold" emphasis` "Confirm Drain" + Button `variant="neutral"` "Cancel". No separate modal.

### /admin/users (Users tab)
- Lists all users with active socket connections (seated + standing)
- Kick confirmation: inline row expands — "Kick [display name]? Their session will end immediately." — Button `variant="fold" emphasis` "Confirm Kick" + Button `variant="neutral"` "Cancel"
- Ban confirmation: inline row expands — "Ban [display name]? They will be unable to join tables." — Button `variant="fold" emphasis` "Confirm Ban" + Button `variant="neutral"` "Cancel"
- Balance grant: BalanceDeltaInput + "Apply" — no confirmation dialog (reversible action; audit log provides traceability)
- Post-action: row updates via delta socket event; no page reload

### /admin/economy (Economy tab)
- StatCard: "Total Chips in Play" (sum of all seated players' chips)
- StatCard: "Active Players" (count of connected users)
- recharts BarChart: chips by table (6 bars), x-axis = table name, y-axis = chips
- No controls on this tab — read-only

### /admin/audit (Audit Log tab)
- Header: "Last 10 Actions"
- Empty state: "No admin actions recorded yet."
- AuditLogRow list, newest first
- No pagination (last-10 only per CONTEXT.md §deferred)
- Auto-refreshes on any mutation event from socket

## Interaction Patterns

### Inline Confirmation Pattern (no modals)
All destructive actions (Kick, Ban, Drain) use in-row expansion instead of a modal overlay. This preserves the admin's spatial context in the list.

1. Admin clicks destructive button (Kick / Ban / Drain)
2. Row smoothly expands downward (CSS max-height transition 200ms ease) to reveal confirmation copy + Confirm + Cancel
3. Confirm triggers socket event + collapses row (or updates row state on response)
4. Cancel collapses row immediately

### Socket-Driven Live State
- No polling. All live data flows via `/admin` namespace push events.
- Loading state on initial connect: show neutral skeleton shimmer (CSS `background: linear-gradient(90deg, rgba(255,255,255,0.03) 25%, rgba(255,255,255,0.07) 50%, rgba(255,255,255,0.03) 75%)` animated) in place of each Card until `adminState` arrives.
- Skeleton width matches expected content width (do not use generic full-width skeleton).

### Form Validation (react-hook-form + zod)
- Validation fires on submit (not on blur) for login form — reduces friction
- Validation fires on blur for inline admin control forms (edit blinds/buy-in, balance delta) — immediate feedback where fields are compact
- Error messages appear directly below the relevant input, 13px, `var(--color-action-fold)`

## Accessibility Contract

- All interactive elements: `minHeight: 44px` minimum (Button primitive enforces this already)
- AdminBanner has `role="banner"` and `aria-label="Admin mode indicator"`
- TabBar has `role="tablist"` (Tab.tsx already sets this on TabBar)
- Confirmation sections have `role="alert"` to announce to screen readers on expand
- Destructive buttons have `aria-label` describing the target: e.g. `aria-label="Kick player Wolf_42"`
- Socket disconnection banner has `role="status"` and `aria-live="polite"`
- Color is never the sole indicator of state — all status values paired with text label (Badge always contains text)

## What Is NOT New UI (Sentry, PostHog, ToS Gate)

These Phase 5 deliverables have zero net-new client UI components:

- **Sentry SDK init** — `client/src/main.tsx` initialization call only. No visible UI. Replay is a background capability.
- **PostHog SDK init** — `client/src/utils/analytics.ts` initialization. `track()` calls are fire-and-forget side effects. No visible UI.
- **ToS gate** — server emits `{ type: 'TOS_REQUIRED' }` error. Existing `Consent.tsx` handles the routing. No new components.

These are excluded from the design contract. The executor should not introduce any new visible UI elements for these three items.
